# 🌱 ○○農業高校 直売所サイト（グループ制作）

わたしたちが育てた作物を紹介する Web サイトです。
グループで分担して作り、最後にインターネットに公開します。

---

## 📁 ファイルと担当

このサイトは「1人1ファイル」で分担します。
自分の担当ファイルだけを編集すれば、ぶつかり（コンフリクト）が起きにくくなります。

| ファイル | なにをする | 担当 |
| --- | --- | --- |
| `index.html` | トップページ（入口・作物の一覧） | 構成リーダー |
| `style.css` | サイト全体の色や形 | デザイン担当 |
| `tomato.html` | 作物ページの**見本**（さわらず残す） | みんなの手本 |
| `template-product.html` | 作物ページの**空テンプレ**（コピーして使う） | ― |
| `自分の作物.html` | 自分の担当作物のページ | 作物担当 |
| `images/` | 写真を入れるフォルダ | 全員 |

---

## ✏️ 編集のしかた

ファイルの中で **`✏️`** がついている所が、書きかえる場所です。
`<!-- -->` で囲まれた緑色の文字（コメント）は説明なので、消さなくてOKです。

### 新しい作物ページの作り方

1. `template-product.html` をコピーする
2. ファイル名を作物名にする（例 `cucumber.html`・ローマ字小文字がおすすめ）
3. `✏️` の所を自分の言葉で書きかえる
4. `images/` に自分の写真を入れて、`<img src="...">` のファイル名を合わせる
5. `index.html` のカードを1つコピーして、リンク先を作ったページ名にする

---

## 🤔 大事なのは「タグ」より「伝え方」

HTML のタグは見本をまねすれば書けます。
このサイトで本当に考えてほしいのは、次の3つです。

1. **だれに買ってほしい？**（その作物を一番喜ぶのはどんな人？）
2. **その人が「ほしい」と思う一言は？**（キャッチコピーは自分の言葉で）
3. **なぜその写真を選んだ？**（一番おいしそうに見える1枚を選ぶ）

---

## 🌐 インターネットに公開する（GitHub Pages）

1. このフォルダを GitHub のリポジトリにアップロードする
2. リポジトリの **Settings → Pages** を開く
3. **Build and deployment → Source** を **Deploy from a branch** にする
4. Branch を `main`、フォルダを `/ (root)` にして **Save**
5. 少し待つと `https://（ユーザー名）.github.io/（リポジトリ名）/` で見られる

> 公開できたら、トップページの URL をグループで共有しよう。
